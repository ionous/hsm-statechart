Data Structure Documentation

struct hsm_info_rec

Notification of important occurances in the state machine.

By design hsm_info only provides information for situtations user code can't sus out via the hsm_machine.h interface. For instance, there isn't a callback on a transition to HsmStateError, since user code can already detect errors via the return codes of HsmProcessEvent() and HsmIsRunning()

typedef void(* hsm_callback_initing)(const hsm_machine hsm, const hsm_state next, void *user_data)

Hear about states just before entering next state.

hsm-statechart supports initial states via the HSM_STATE macros.

Parameters:
hsmThe hsm_machine. note: hsm->current has the new state, and hsm->stack.context has most recently pushed context.
nextThe hsm_state that's about to be entered.
user_dataThe hsm_info_rec.user_data.
Note:
The sequence in UML is enter first, take the init transition second.

void HsmSetInfoCallbacks ( hsm_info_t *  callbacks,
hsm_info_t *  old_callbacks 
)

Install a new set of callbacks for spying on internal statemachine processing.

Currently there's just one set of callbacks for all threads; this is not locked or synchronized in anyway.

Parameters:
callbacksThe set of callbacks; individual callbacks can be NULL, as can the callbacks pointer itself.
old_callbacksThe last set of callbacks passed to HsmSetInfoCallbacks. Its recommended that you should record, and later restore the old callbacks.
